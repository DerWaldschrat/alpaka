43 Fibers
44 ------
45 
46 A fiber is a user-space thread with cooperative context-switch.
47 They are implemented on top of coroutines. A coroutine is a function that can be suspended and resumed but has not necessarily a stack.
48 boost::fiber = stackful coroutine + scheduler + sync (no wait â†’ next fiber in thread)
49 C++17: N3858, N3985, (N4134 stackless coroutines 'await')
50 
51 ### Positive
52 * Less cost of creation.
53 * Less cost of thread change.
54 * Deterministic thread change.
55 * No locks at all (Always only one active fiber per kernel-thread).
56 * Prevents false sharing because all fibers working on nearby values are in the same block and can be executed by the same kernel-thread on the same core.
57 * Prevents cache thrashing (threads on the same core compete for the same cache line) by using a user-level scheduler for the fibers that can invocate them in order of access of the memory.
